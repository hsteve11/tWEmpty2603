Honestly, no. Each new chat starts with a blank slate -- I don't carry memory between conversations.

**What you can do:**

Drop a short note at the start of a new chat, something like:

> "When making changes, give me diffs/patches instead of rewriting the whole file. Only do full rewrites when changes are too scattered."

That one sentence is enough to set the pattern for the whole session.

**If you want something more permanent**, you could keep a small text file (like `ai-prefs.txt`) with your preferences:

```
- Prefer diffs over full file rewrites
- Only full rewrites when changes are widespread
- Default dark themes
- Use DM Mono + Syne font pairing
```

Then paste it in at the start of a new project. Takes 5 seconds and saves you repeating yourself.

**Git actually helps here too** -- if you have the repo set up, you can paste a diff into the chat and say "apply this change" or "revert this," and I can work from that context without needing the full file history.

It's a limitation of how I work, but the workaround is lightweight enough that it shouldn't slow you down.
